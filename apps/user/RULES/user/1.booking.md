# User Rules & Workflows

Guidelines for authenticated patients.

## 1. Booking & The "Bundle" System
- **Steps**: 5-Step Wizard (Service ➔ Date & Time ➔ Other Info ➔ Review ➔ Confirm).
- **Primary + Waitlist**: Users can book a "Primary" appointment (secured slot) and a "Waitlist" choice (preferred slot) in a single atomic transaction.
- **Book for Others**:
    - Users can book on behalf of family/friends.
    - If "Book for Self" is selected, `booked_for_name_parts` is sent as `null`, and the backend snapshots the user's profile data.
    - If "Book for Someone Else" is selected, the specific names provided are snapshotted.

## 2. Universal Pending Approval
- **Status Upon Booking**: Every appointment (both General and Specialized) starts with a `PENDING` status and `approval_status: 'pending'`.
- **Clinic Review**: A receptionist must manually review and approve the request before it is finalized as `CONFIRMED`.

## 3. Waitlist Strategy
- **Just Waitlist**: Users can join the waitlist without a primary appointment if no slots are free.
- **Bundled Waitlist**: If the user has a primary appointment, claiming a waitlist slot **automatically cancels** the primary appointment (Swap Logic).
- **Global Constraints**: Enforces the 3-entry limit and 1-per-day-per-service cap.

## 4. Restriction Rules (Sync)
- **Status Check**: The system checks `is_booking_restricted` before allowing a user to join a waitlist or book a slot.
- **Advance Booking Window**: Restricted users are limited to a **5-day advance window** (`CLINIC_CONFIG.NO_SHOW_RESTRICT_ADVANCE_DAYS`).
- **Auto-Unlock**: Restrictions are automatically lifted if the current date is past `restriction_until`.
