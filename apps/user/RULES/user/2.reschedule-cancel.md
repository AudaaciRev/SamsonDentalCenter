# User Actions: Reschedule & Cancel

Rules for managing existing appointments.

## 1. Rescheduling Policy
- **One-and-Done**: Patients are limited to **ONE** reschedule per appointment. The "Reschedule" button is disabled once the `reschedule_count` reaches 1.
- **Availability Sync**: Rescheduling triggers a new search for available slots, respecting the same Dentist Assignment rules as a fresh booking.
- **Dentist Preference**: If the original appointment was "Preferred," the system attempts to default the reschedule to the same dentist if they are free.

## 2. Cancellation Policy
- **Notice Windows**:
    - **≥ 24 Hours**: Status becomes `CANCELLED`. No strikes recorded.
    - **< 24 Hours**: Status becomes `LATE_CANCEL`. Counts toward booking restrictions.
- **Cascade Response**: Cancelling an appointment automatically triggers the **Waitlist Notification** system if there is at least 3 hours' notice.

## 3. Notification Logic
- **Real-Time Sync**: Rescheduling or Cancelling triggers an immediate backend notification to the clinic's management dashboard.
- **Email Receipt**: Patients receive a "Reschedule Confirmation" or "Cancellation Receipt" email for all successfully processed actions.
